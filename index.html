Divvi

A mobile dApp built on the Ethereum blockchain. Divvi uses a custom coin, called Divvi Coin, for all transactions made within the application. Users earn coins by watching ads and then distribute those coins to charities of their choosing. Smart contracts were built, compiled, and deployed using the Truffle framework and the Solidity language. The mobile application was build using React Native + Redux and Web3 to implement JSON-RPC and interact with the smart contracts.

Browse Organizations

The main page is styled in a Tinder-like manner - users swipe left or right to browse different charities. Embedded in the stack of charities, there are advertisements with associated coin values. Clicking or swiping right on an ad opens a webview to display the ad. The webview closes once the ad completes and the user's balance increases by the associated coin value. Swiping right on a charity donates to that organization.

Manage Account

The account tab shows the current balance, the total amount donated, and the current donation amount. The donation amount determines how many coins are donated to a charitable organization when the user swipes right and may be updated. The history tab lists all donations and the ads watched. Users can share history via text/email/social media.

Technology

Blockchain: Truffle, Solidity, MetaMask, Geth, TestRPC, Mocha + Chai

Mobile: React Native, Redux, Web3, NativeBase

Back End: NodeJS, Express, PostgreSQL

Budgie

Budgie is a mobile application that uses Tesseract OCR to convert receipt images into usable data. This data is then visualized using D3/NVD3 to show spending patterns. The back end is hosted on an AWS EC2 instance, running a NodeJS + Express server. The mobile application was built using AngularJS, Ionic, and Cordova.

Front End Back End View Demo

Image Recognition

Budgie allows users to take a picture of a receipt using their device's camera or select an image from their camera roll. A base64 encoding of the image is then sent to the server, which uses an ImageMagick script to clean the image. The cleaned TIF file is then processed by Tesseract OCR to extract the text. Regualar expressions parse the text string to return individual receipt items. These items are then sent back to the application as JSON. The user is then able to edit the populated data before it is stored in the database.

Data Visualization

Budgie makes use of the NVD3 library to create D3 data visualizations, based on the user's spending data. The data is retrieved from the database, formatted into an NVD3-friendly format, then rendered into an interactive graph that allows user's to analyze their spending over time and across different categories. Users can toggle categories and locations on and off to view potential savings over time.

Technology

Mobile: Angular, Ionic, Cordova, D3, NVD3

Back End: NodeJS, Express, PostgreSQL, Tesseract OCR, ImageMagick

Hosting: AWS - EC2 instance

nutriSNP

nutriSNP is an advanced calorie and macronutrient calculator that integrates with the 23andMe API to determine users' metabolic phenotype. This phenotype, along with some basic lifestyle data, is used to calculate caloric and macronutrient reccommendations based on the user's current goals. The raw genetic data is sent to the server, which analyzes the SNPs and returns the user's phenotype. This phenotype is then stored in the database and used to calculate calorie and macronutrient recommendations any time the user changes their profile data.

Genetic Data

Genetic data is gathered via the 23andMe API. The user must authenticate nutriSNP to import their results the first time they login. Those results are then used to calculate the user's metabolic phenotype. Both the genotype and phenotype are displayed on the user's profile page and saved to the database.

Calorie Recommendations

Calorie recommendations are based on user-entered lifestyle data (height, weight, age, gender, and activity levels). The macronutrient ratios and amounts are determined from the user's 23andMe results. These recommendations are then displayed on a clean, easy-to-read profile page. If a user does not currently have an account with 23andMe, they may still receive calorie and macronutrient recommendations based on their lifestyle data.

Technology

Front End: Javascript, EJS, Materialize, PassportJS

Back End: NodeJS, Express, PostgreSQL, Mocha + Chai, Heroku, 23andMe API

Realist

Realist is a drawing game that makes use of the Google Vision API. Users are prompted to draw a specific image. Once the image is submitted, Google Vision attempts to guess the prompt. On the results page, users can browse submissions from other users, along with the Google Vision results. Users may log in with either Google or Facebook to sign their artwork. Otherwise, the drawings will show as Anonymous.

View Code Live Site

Image Recognition

Once a user submits an image, the base64 string is sent to the Google Vision API for interpretation and the user is taken to the results page. The results are listed in order of confidence percentage. Generic results, such as "line art" are filtered from the results. The submitted image is also sent to the Firebase database to later be displayed in the gallery.

Gallery

The gallery shows artwork submitted by users, along with the corresponding Google Vision results. If a user logs in, their name is displayed with their drawing. Otherwise, the images are shown as annonymous.

Technology

Javascript, Materialize, Firebase, Google Vision API, DrawingBoard.js
